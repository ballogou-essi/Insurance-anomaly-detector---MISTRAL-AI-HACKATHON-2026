# Insurance Anomaly Detector – AI-Assisted Anomaly Detection

## Business Objective

Démo: https://youtu.be/XMHz59fp5-I?si=ZsLREIBCfXg-yI-f

This project simulates a **decision-support tool** for the analysis of insurance underwriting files.

In a real-world scenario, an insurer receives files (PDFs, forms, supporting documents, CSV/JSON exports). 
The goal is to **quickly identify inconsistencies** or risk signals in order to:
- accelerate the processing of straightforward cases,
- trigger a **manual review** when a file is ambiguous,
- flag potentially fraudulent or abnormal cases for escalation.

The system produces a standardized business-oriented output:

- **status**
  - `OK`: file is consistent, no notable anomaly
  - `REVIEW`: file is ambiguous/incomplete, requires verification
  - `FLAGGED`: strong inconsistencies/high risk, escalation recommended
- **risk_score (0..100)**: estimated risk level
- **top_alerts**: 3 to 7 short alerts (detected signals)
- **recommended_action**: recommended action (Approve / Review / Escalate)
- **summary_explanation**: a clear, business-oriented synthesis

> Important: This is an **AI-assisted analysis**, it is **not** a final human validation.

## How It Works (Pipeline)
1. **Input**
   - Upload a file (PDF/TXT/CSV/JSON) via the web interface
   - OR quick entry via a form (fallback option)

2. **Extraction & Normalization**
   - Text extraction (PDF → text)
   - Normalization to provide stable input for the model

3. **AI Analysis (Mistral)**
   - Mistral receives the content and returns a strict business JSON format:
     `status`, `risk_score`, `top_alerts`, `recommended_action`, `summary_explanation`
   - *Note: The input data for demonstrations or tests is generated by an AI.*

4. **Results Display**
   - Business-oriented display (badge, score, alerts, action, summary)
   - Technical JSON available in a collapsible section

5. **PDF Export**
   - A PDF report is generated server-side, containing:
     status, score, alerts, action, summary, and an "AI-assisted analysis" disclaimer
